
## 1.1 什么是数据结构

### 1.1.1 数据结构的基本含义
数据结构是计算机中组织和存储数据的方式，它不仅仅是简单地把数据放一起，还包括数据之间的关系，以及对这些数据能进行的操作。

简单来说：
- **数据**：是描述客观事物的符号，比如数字、文字、图像等。
- **数据项**：数据的最小单位，比如一个学生的“学号”就是一个数据项。
- **数据元素**：数据的基本单位，通常由若干数据项组成，比如一个“学生”记录包括学号、姓名、成绩等数据项。
- **数据对象**：是具有相同性质的数据元素的集合，比如所有学生的记录。
- **数据结构**：是相互之间存在一种或多种特定关系的数据元素的集合。

数据结构研究的核心有三个方面：
- **逻辑结构**：数据元素之间的逻辑关系（抽象的关系）。
- **存储结构**（物理结构）：数据在计算机内存中的表示方式。
- **数据的运算及其实现**：对数据能做的操作，以及如何实现这些操作。

### 1.1.2 逻辑结构 vs 存储结构
**逻辑结构**描述数据之间的关系，与存储方式无关，关注“是什么”。常见类型后面会详细说。

**存储结构**是逻辑结构在计算机中的实际存放形式，主要有两种：
- **顺序存储**：用一组连续的存储单元依次存放，比如数组。
- **链式存储**：用节点存放数据元素和指针指向下一个节点，比如链表。

举例：
- 线性表（如班级学生名单）：逻辑上是“一个接一个”的线性结构；可以用数组（顺序存储）实现，也可以用链表（链式存储）实现。
- 树（如家族族谱）：逻辑上是“父子关系”；存储时可以用数组（孩子兄弟表示法）或链表。
- 图（如城市地铁线路）：逻辑上是“任意两点可能相连”；存储常用邻接矩阵（二维数组）或邻接表（链表）。

### 本节小结/核心考点
- 数据结构是带关系的数据元素的集合，包括逻辑结构和存储结构。
- 数据 → 数据项 → 数据元素 → 数据对象 → 数据结构（层层包含）。
- 逻辑结构关注关系，存储结构关注内存表示（顺序 vs 链式）。
- 数据结构研究三方面：逻辑结构、存储结构、运算及其实现。
- 常见例子：线性表（顺序/链式）、树、图。

## 1.2 基本概念和术语

### 1.2.1 常见术语解释
- **数据**：能输入到计算机并被处理的符号总称。
- **数据元素**：数据的基本单位，在程序中通常作为一个整体考虑。
- **数据项**：构成数据元素的不可分割的最小单位（相当于字段）。
- **数据对象**：性质相同的数据元素的有限集合。
- **数据类型**：一个值的集合 + 对这个集合的一组操作（如int、float）。
- **构造类型**（也叫数据结构）：由若干数据元素按某种关系组织而成（如数组、链表、栈等）。

### 1.2.2 逻辑结构的四种基本类型
- **集合结构**：数据元素之间除了“同属一个集合”外，别无其他关系。  
  例子：图书馆的全部图书，彼此没有直接关系。
- **线性结构**：元素之间是一对一的关系（一个前驱、一个后继）。  
  例子：队列、栈、线性表（如学生按学号排的名单）。
- **树形结构**：元素之间是一对多的层次关系。  
  例子：计算机文件夹系统、公司组织架构。
- **图状结构**：元素之间是多对多的任意关系。  
  例子：社交网络、城市交通地图。

### 1.2.3 术语对比表格

| 术语         | 含义                                     | 与其他术语关系                     |
|--------------|------------------------------------------|------------------------------------|
| 数据项       | 最小单位，有独立含义                     | 组成数据元素                       |
| 数据元素     | 基本单位，通常整体处理                   | 由数据项组成，属于数据对象         |
| 数据对象     | 同性质数据元素的集合                     | 是数据结构的处理对象               |
| 数据类型     | 值集合 + 操作集合                         | 语言层面，构造类型是其一种         |
| 数据结构     | 带关系的数据元素集合                     | 包括逻辑结构和存储结构             |

### 本节小结/核心考点
- 数据元素是基本单位，数据项是最小单位，数据对象是同类元素的集合。
- 逻辑结构四种：集合（无关系）、线性（一对一）、树形（一对多）、图状（多对多）。
- 线性结构最常见，如线性表、栈、队列。
- 记住术语层次：数据项 → 数据元素 → 数据对象 → 数据结构。
- 区分数据类型（语言层面）和数据结构（组织方式）。

## 1.3 抽象数据类型（ADT）的表示与实现

### 1.3.1 什么是抽象数据类型（ADT）

- **核心思想**：只描述“做什么”（外部接口/操作），隐藏“怎么做”（内部实现）
- 定义：一个数学模型 + 定义在其上的操作集合
- 类比：开车只用方向盘、油门、刹车，不需要懂发动机原理

### 1.3.2 ADT 的标准三元组描述

ADT = (D, S, P)

- **D（Data Object）**：数据对象的定义（元素取值范围）
- **S（Data Relations）**：数据元素之间的关系
- **P（Basic Operations）**：基本操作集合

### 1.3.3 示例：ADT Complex（复数）

- **ADT 名称**：Complex
- **D**：{ <e1, e2> | e1, e2 ∈ Real }（有序对：实部 + 虚部）
- **S**：有序对关系
- **P**（部分操作）：
    - InitComplex(&Z, v1, v2)：构造复数
    - GetReal(Z)：取实部
    - GetImag(Z)：取虚部
    - Add(Z1, Z2)：返回 Z1 + Z2
    - Sub/Zul/Mul 等类似

### 1.3.4 表示 vs 实现

- **表示**：逻辑层面用三元组或伪代码描述（数学模型）
- **实现**：编程层面选择存储结构 + 代码实现操作
    - 同一 ADT 可有多种实现（如线性表的顺序实现 vs 链式实现）

### 本节小结/核心考点

- ADT 核心：数据抽象 + 操作抽象（只关心接口）
- ADT 三元组：(D, S, P)
- 表示（逻辑描述） ≠ 实现（代码实现）
- 同一 ADT 可多种存储实现（体现灵活性）

## 1.4 算法和算法分析

### 1.4.1 什么是算法
算法是对特定问题求解步骤的精确描述，是一系列有顺序的操作指令。

一个好的算法必须具备五个特性：
- **有穷性**：有限步骤后必须结束，不能无限循环。
- **确定性**：每一步都有确切含义，不会出现二义性。
- **可行性**：每一步都能通过有限次基本操作实现。
- **输入**：有零个或多个输入。
- **输出**：至少有一个输出（结果）。

### 1.4.2 算法好坏的衡量标准
- **正确性**：能正确解决问题的算法。
- **可读性**：代码易懂、易维护。
- **健壮性**：对非法输入能适当处理，不崩溃。
- **效率**：时间效率高（运行快）、空间效率高（占用内存少）。

其中时间和空间效率最常考。

### 1.4.3 时间复杂度和空间复杂度
- **时间复杂度**：算法执行所需的基本操作次数的量级，随问题规模 n 的增长趋势。
- **空间复杂度**：算法运行时占用的内存空间量级。

我们用**大 O 记号**表示渐近时间复杂度（忽略常数和低阶项，只看最高阶）：
- O(1)：常数时间，如访问数组元素。
- O(n)：线性时间，如遍历一次数组。
- O(n²)：平方时间，如冒泡排序、简单矩阵乘法。

为什么用渐近复杂度而不是具体运行时间？
因为具体运行时间受机器速度、编译器、语言等影响，而渐近复杂度只看算法本身效率，与硬件无关，能客观比较不同算法。

### 本节小结/核心考点
- 算法五特性：有穷性、确定性、可行性、输入、输出。
- 算法评价：正确性、可读性、健壮性、时间效率、空间效率。
- 时间复杂度用大 O 表示算法随 n 增长的趋势。
- 常见复杂度：O(1) < O(n) < O(n²)。
- 用渐近复杂度是因为它与具体机器无关，能本质反映算法效率。
- 空间复杂度同样关注额外内存占用量级。
